How This Website Works

From Cognito auth to DynamoDB leaderboard updates

portfolio breakdown

Finance Clicker is a full-stack game where every important action is validated in AWS. Instead of storing score logic only in-browser, the app routes scoring and upgrade purchases to API endpoints backed by Lambda so players cannot edit local variables to cheat points.

1) Authentication (AWS Cognito)

Users log in through Amazon Cognito Hosted UI. After sign-in, the app receives tokens through the OAuth callback URL and uses those tokens for authorized requests. This ties every score update to a verified user.

2) Gameplay Requests (API Gateway + Lambda)

When you click the trade button or buy upgrades, the frontend calls API Gateway routes such as /click, /buy, and /state. Lambda functions apply server-side game rules for points-per-click and points-per-second, then return updated player state.

3) Persistent Data (DynamoDB)

Player progression, upgrade levels, and cumulative points are stored in DynamoDB. Because this state lives in a managed database, users keep progress between sessions and devices. Leaderboard queries read top scores from DynamoDB and render rankings live in the UI.

4) Frontend Design Choices

The main page uses a finance terminal aesthetic with ticker tape, market-inspired visuals, and glassy panels. This page keeps the same style language while switching to a "movie terminal" background to reinforce the engineering narrative behind the project.

TL;DR: Cognito identifies users, Lambda enforces game rules, DynamoDB stores progression, and the frontend visualizes state with a finance-terminal style.
